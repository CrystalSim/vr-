# 大作业包装说明

## 项目定位

本项目不是单纯复现一个摘要算法，而是把 360°视频摘要论文中的三阶段系统做成一个可运行的课堂演示系统：

1. 从 360°视频中生成 saliency maps。
2. 根据显著区域自动裁剪普通 2D event video。
3. 再对 event video 做时间摘要，导出最终短 2D summary video。

这样答辩时可以明确说明：CA-SUM-360 / S³-360 是算法基础，我们的工作是把论文流程工程化、可视化、可交互化，并补齐最终可播放结果。

## 论文对齐增强

原论文的关键链路包含“静态/运动相机判别”和“spatio-temporal sub-volume 形成”。项目现在加入了对应的轻量实现：

- 相机运动诊断：分析 ERP 帧顶部和底部区域的帧间变化，判断输入视频更接近静态相机还是运动相机，并给出置信度。
- Saliency 路径解释：根据相机类型解释应采用更适合运动相机或静态相机的 saliency 分支，呼应论文中 ATSal / SST-Sal 的选择机制。
- Event sub-volume 诊断：把高显著片段按时间邻近和推荐视角邻近聚合，形成事件级子体，并输出事件数量、覆盖率、持续时间和平均显著性。

## 新增交互体验

项目加入了一个类 YouTube 的 360°原视频播放器：上传 MP4 后，页面可以直接把原始等距矩形视频映射成可拖拽视角的全景播放画面。播放器支持播放/暂停、拖拽视角、滚轮缩放、全屏，并把 S3-360-TourGuide 识别出的导览点显示为可点击章节。

这部分不是替代摘要算法，而是把算法结果和观看体验连接起来：用户既能像看 YouTube 360°视频一样自由探索，也能开启摘要章节巡航，让系统自动跳转到关键片段并贴合推荐视角。

当前扩展优先面向普通浏览器演示，不依赖 VR 头显。播放器会记录用户观看过程中的 yaw、pitch、FOV、当前模式和推荐视角误差，并可导出 CSV。系统还会展示推荐导览路径的平均转向角、最大转向角、平均转向速度和路线状态，用来说明摘要结果是否适合连续观看。

## 三阶段流程

### Step 1: Saliency Maps

输入：360° ERP 视频帧。

输出：每一帧的显著性热力图，表示用户最可能关注的区域。

项目中体现为：

- 原始全景帧展示。
- saliency heatmap 展示。
- 推荐视角框展示。

### Step 2: 2D Event Video

输入：原始 360°视频帧和显著性热力图。

处理：

- 找出高显著性区域。
- 按片段聚合 event。
- 围绕推荐视角裁剪普通 16:9 画面。

输出：`step2_2d_event_video.mp4`。

这个输出已经不是 360°视频，而是普通屏幕可以直接观看的 2D event video。它保留所有重要事件，但还没有做最终时间摘要。

### Step 3: Final Output

输入：Step 2 得到的 event 片段。

处理：使用 CA-SUM / S³-360 风格的时间摘要方法，从 event video 中选出最关键、少重复、覆盖事件更完整的片段。

输出：

- `step3_final_summary.mp4`：最终短 2D 视频。
- `s3_360_summary.gif`：带热力图和视角框的可视化动图。
- `s3_360_tourguide_report.md`：面向答辩展示的导览说明。

## 相比复现的提升

- 增加了完整三阶段可视化页面，而不是只输出指标。
- 增加了轻量相机运动判别和 event sub-volume 诊断，更贴近原论文的完整系统流程。
- 增加了类 YouTube 的 360°原视频播放器，支持原视频直接全景播放和摘要章节巡航。
- 增加了 S3-360-TourGuide，将摘要片段识别为导览点，并生成有顺序的智能导览路线。
- 增加了浏览器内观看轨迹记录、CSV 导出、推荐视角误差和路线状态展示。
- 增加了 2D event video 这个中间结果，便于解释论文流程。
- 增加了最终短 2D MP4 导出，答辩时可以直接播放。
- 增加了摘要关键帧 360°/VR 巡航，可验证推荐视角是否合理。
- 保留严格实验脚本，在有人工标注时再输出论文实验指标；网页 Demo 默认不展示无真值支撑的评分。
- 支持上传真实 360°视频，也支持 NPZ/HDF5 实验数据。

## 推荐答辩演示顺序

1. 先说明痛点：360°视频内容多，用户不知道看哪里，也不想完整看完。
2. 展示相机运动与事件子体诊断：说明系统不只是抽帧，而是对齐论文中的相机分支和事件建模。
3. 先展示 YouTube-style 360°原视频播放器：说明系统支持直接播放和自由探索原始 360°视频。
4. 展示摘要章节巡航：点击摘要章节或开启“Σ”，说明算法结果可以驱动自动导览。
5. 展示观看轨迹与路线状态面板：说明系统能记录用户视角，并说明推荐导览是否平滑。
6. 展示 Step 1：系统先判断每一帧哪里重要。
7. 展示 Step 2：系统把 360°视频裁剪成普通 2D event video。
8. 展示 Step 3：系统再做时间摘要，导出最终短 2D 视频。
9. 展示导览报告下载：说明系统可以把导览点、推荐方向和路线状态导出为答辩材料。

## 一句话总结

我们复现了 360°视频时空摘要的核心流程，并进一步做成了一个可以上传视频、诊断相机运动、构建事件子体、直接全景播放原视频、记录观看轨迹、展示导览路线状态、查看显著性、生成 2D event video、导出最终短视频、用摘要章节自动导览推荐视角的完整系统。
