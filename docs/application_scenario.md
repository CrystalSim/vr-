# 面向沉浸式校园/景区导览的 360°视频智能摘要与观看引导系统

## 1. 应用背景

随着 360°相机和全景视频平台的发展，校园开放日、博物馆展厅、景区步道、实验室参观、城市街区漫游等场景中越来越多地使用 360°视频进行沉浸式展示。相比普通视频，360°视频能够保留完整空间信息，用户可以自由拖拽视角，从不同方向观察环境。

但在实际观看中，360°导览视频也带来新的问题：

- 视频通常较长，用户完整观看成本较高。
- 用户虽然可以自由转动视角，但不知道关键内容出现在哪个方向。
- 普通视频摘要只解决“看哪一段”，没有解决“在 360°画面里看哪里”。
- 如果推荐视角频繁大幅跳变，连续观看时容易产生不适感。
- 缺少对用户观看轨迹的记录与分析，难以判断用户是否真正看到了重点区域。

因此，本项目将 360°视频摘要任务放到“沉浸式导览”场景中，目标不只是生成短视频，而是生成一条可观看、可引导、可评价的智能导览路线。

## 2. 场景定位

本项目面向的典型场景是：

> 用户上传一段较长的校园、景区或展厅 360°导览视频，系统自动提取关键导览片段，为每个片段生成推荐观看方向，并在浏览器 360°播放器中引导用户观看重点区域，同时记录用户观看轨迹并评价导览舒适度。

在该场景中，系统需要同时回答三个问题：

1. 哪些时间片段值得看？
2. 每个片段中应该看向哪个方向？
3. 这条导览路线是否平滑、用户是否跟上了推荐视角？

## 3. 系统目标

围绕沉浸式导览场景，系统目标可以概括为：

- 从长 360°视频中自动提取关键导览点。
- 为每个导览点生成推荐观看视角。
- 将摘要结果组织成一条可连续观看的导览路线。
- 在浏览器 360°播放器中提供自动导览、摘要巡航和自由观看模式。
- 记录用户观看轨迹，分析用户视角与推荐视角的偏差。
- 评价导览路线的内容覆盖率和观看舒适度。

## 4. 与现有系统的对应关系

当前系统中的模块可以自然映射到该应用场景：

| 当前系统模块 | 场景化解释 |
| --- | --- |
| Saliency Maps | 估计导览视频中值得关注的位置 |
| 片段摘要 | 提取关键导览点 |
| 推荐视角 | 为每个导览点生成推荐观看方向 |
| S3-360-Guide | 面向导览路线的摘要与视角优化方法 |
| 事件覆盖率 | 导览点/景点/展区覆盖程度 |
| 视角跳变 | 导览路线是否平滑 |
| 观看轨迹 CSV | 用户观看行为记录 |
| 舒适度分数 | 连续观看体验评价 |

因此，本项目不是单纯复现视频摘要算法，而是将算法应用到 360°导览任务中，形成“摘要 + 视角推荐 + 交互观看 + 轨迹分析 + 舒适度评价”的完整流程。

## 5. 推荐优化点

### 5.1 导览点识别

将原来的“摘要片段”进一步包装为“导览点”：

```text
摘要片段 -> 导览点
推荐视角 -> 推荐观看方向
事件覆盖率 -> 景点/展区覆盖率
舒适度 -> 导览路线平滑度
```

页面中可以将选中的片段展示为：

- 导览点 1：入口或开场区域
- 导览点 2：主要建筑、展品或景观
- 导览点 3：人群互动或关键事件
- 导览点 4：结束视角或全景收束

这样可以让系统更贴近真实导览应用，而不只是显示抽象的片段编号。

### 5.2 智能导览路线生成

在导览场景中，摘要结果不应只选择分数最高的片段，还应形成一条适合连续观看的路线。可将路线评分设计为：

```text
RouteScore =
  内容重要性
+ 景点/事件覆盖
+ 视觉新颖性
+ 推荐视角稳定性
- 重复内容
- 大角度转头
```

当前的 `S3-360-Guide` 可以解释为面向导览场景的改进方法：它在内容重要性之外加入事件覆盖和视角稳定性，使导览路线既覆盖更多内容，又减少观看过程中的突兀转向。

后续可以进一步扩展一个场景化方法名：

```text
S3-360-TourGuide
```

该方法可以专门优化“导览点覆盖率”和“转向舒适度”。

### 5.3 导览地图式展示

为了让答辩展示更直观，可以将推荐导览路线直接画在 ERP 全景图上：

- 用编号点表示系统选出的导览点。
- 用箭头表示推荐观看顺序。
- 标注每个导览点的时间和片段编号。
- 展示从一个导览点切换到下一个导览点需要转向的角度。
- 用状态标签显示导览路线是否平滑。

这种展示方式比单纯的 yaw/pitch 坐标图更容易理解，适合作为系统主展示页。

### 5.4 用户观看轨迹分析

播放器可以记录用户观看过程中的视角轨迹，并与推荐视角进行对比。推荐分析指标包括：

- 平均视角误差：用户视角与推荐视角的平均夹角。
- 最大偏离片段：用户最没有跟上推荐视角的片段。
- 推荐视角命中率：误差小于阈值的时间比例。
- 关键导览点跟随率：用户是否在每个导览点看向推荐区域。
- 自由探索比例：用户主动偏离推荐路线的程度。

通过这些指标，系统可以从“记录轨迹”升级为“评价导览效果”。

### 5.5 导览舒适度评价

360°导览不只关注内容，还要关注观看体验。推荐保留并强化以下舒适度指标：

- 平均转向角
- 最大转向角
- 平均转向速度
- 视角跳变次数
- 舒适度分数

这些指标可以说明系统生成的摘要是否适合连续观看，避免用户频繁大幅转头。

### 5.6 自动导览报告

为了形成完整工程闭环，可以增加自动报告导出功能。报告内容包括：

```text
视频总时长
摘要后导览时长
导览点数量
景点/事件覆盖率
平均转向角
最大转向角
平均转向速度
舒适度分数
用户平均视角误差
用户跟随率
```

该报告可作为实验结果，也可直接用于课程答辩展示。

## 6. 推荐答辩表述

可以用下面这段话概括项目定位：

> 本项目面向校园和景区等沉浸式导览场景，设计并实现了一个 360°视频智能摘要与观看引导系统。系统从长 360°视频中自动提取关键导览点，生成推荐观看方向和导览路线，并通过浏览器播放器记录用户观看轨迹，评价导览路线的内容覆盖率、视角平滑度和用户跟随效果。相比普通视频摘要，本系统不仅回答“看哪一段”，还进一步回答“在 360°画面中看哪里”以及“这条导览路线是否适合连续观看”。

## 7. 后续优先级建议

建议按照以下顺序继续扩展：

1. 增加用户观看轨迹分析指标，例如平均视角误差、推荐命中率和用户跟随率。
2. 增加导览报告导出功能，生成可用于答辩的 `report.md` 或 `report.html`。
3. 将摘要片段命名为“导览点”，优化页面文案和展示逻辑。
4. 新增 `S3-360-TourGuide` 方法，进一步强化导览点覆盖和转向舒适度。
5. 增加 A/B 对比页面，对比原始摘要方法和导览优化方法的路线差异。
