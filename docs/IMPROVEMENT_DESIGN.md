# S3-360-Guide 改进方案与设计

## 1. 改进目标

原 360°视频摘要方法主要解决“从长视频中选哪些片段”的问题，但 360°视频还有一个额外难点：用户即使看到了被选中的片段，也未必知道应该看全景画面的哪个方向。

本项目在原 S3-360 的基础上提出 `S3-360-Guide`，目标是把摘要结果从“关键片段集合”进一步提升为“关键片段 + 推荐观看导览”的展示结果。

## 2. 核心创新点

### 2.1 事件覆盖增益

如果摘要只根据显著性或重要性选择，容易反复选择同一类画面。改进方法在贪心选段时加入事件覆盖增益：

- 若候选片段属于尚未覆盖的新事件，则获得额外奖励。
- 若该事件已被选中过，则奖励降低。
- 这样可以让摘要覆盖更多不同内容。

### 2.2 视角稳定性约束

360°视频中显著区域可能快速移动，如果摘要选中的片段视角变化太剧烈，普通屏幕观看会不舒服。改进方法计算每个片段附近推荐视角中心的局部变化：

- 推荐视角变化越小，稳定性越高。
- 稳定片段更适合生成普通屏幕导览摘要。
- 该项用于提升观看舒适度。

### 2.3 导览型综合评分

改进方法使用如下综合评分：

```text
Score(i) =
  alpha * Saliency(i)
+ beta  * Importance(i)
+ gamma * Novelty(i)
+ delta * Continuity(i)
+ eta   * EventGain(i)
+ theta * ViewStability(i)
- lambda * Redundancy(i)
- mu     * ViewJump(i)
```

各项含义：

- `Saliency`：画面是否吸引注意。
- `Importance`：内容是否重要。
- `Novelty`：是否提供新信息。
- `Continuity`：与已选片段是否连贯。
- `EventGain`：是否覆盖了新事件。
- `ViewStability`：推荐视角是否稳定，适合普通屏幕观看。
- `Redundancy`：是否和已选内容重复。
- `ViewJump`：推荐视角跳变是否过大。

## 3. 相比原方法的可讲点

原 S3-360 更关注摘要片段的选择，`S3-360-Guide` 更强调观看体验：

```text
原方法：选择关键片段
改进方法：选择关键片段 + 覆盖更多事件 + 控制视角跳变 + 输出更适合导览的视频
```

展示时可以这样说明：

> 本项目在原有显著性、重要性和多样性约束的基础上，进一步加入事件覆盖增益和视角稳定性约束，使生成的摘要不仅内容更丰富，而且观看过程更平滑，更适合普通屏幕导览。

## 4. 项目落地方式

代码中新增 `S3-360-Guide` 方法，并加入网页 Demo 的方法选择列表。用户可以直接在页面中切换 `S3-360` 和 `S3-360-Guide`，观察摘要片段、推荐视角框、时间轴和方法对比的变化。

## 5. 适合答辩展示的总结

本改进不是简单增加界面功能，而是围绕 360°视频的观看特点加入导览约束。它解决了两个问题：

1. 摘要内容是否覆盖足够多的事件。
2. 摘要视频是否适合普通用户连续观看。

因此，`S3-360-Guide` 可以作为在原论文方法基础上的应用型创新点。
