# S3-360-Guide 改进方案与设计

## 1. 改进目标

原 360°视频摘要方法主要解决“从长视频中选哪些片段”的问题，但 360°视频还有一个额外难点：用户即使看到了被选中的片段，也未必知道应该看全景画面的哪个方向。

本项目在原 S3-360 的基础上提出 `S3-360-Guide`，目标是把摘要结果从“关键片段集合”进一步提升为“关键片段 + 推荐观看导览”的展示结果。

## 2. 核心创新点

### 2.1 事件覆盖增益

如果摘要只根据显著性或重要性选择，容易反复选择同一类画面。改进方法在贪心选段时加入事件覆盖增益：

- 若候选片段属于尚未覆盖的新事件，则获得额外奖励。
- 若该事件已被选中过，则奖励降低。
- 这样可以让摘要覆盖更多不同内容。

### 2.2 视角稳定性约束

360°视频中显著区域可能快速移动，如果摘要选中的片段视角变化太剧烈，普通屏幕观看会不舒服。改进方法计算每个片段附近推荐视角中心的局部变化：

- 推荐视角变化越小，稳定性越高。
- 稳定片段更适合生成普通屏幕导览摘要。
- 该项用于提升观看舒适度。

### 2.3 导览型综合评分

改进方法使用如下综合评分：

```text
Score(i) =
  alpha * Saliency(i)
+ beta  * Importance(i)
+ gamma * Novelty(i)
+ delta * Continuity(i)
+ eta   * EventGain(i)
+ theta * ViewStability(i)
- lambda * Redundancy(i)
- mu     * ViewJump(i)
```

各项含义：

- `Saliency`：画面是否吸引注意。
- `Importance`：内容是否重要。
- `Novelty`：是否提供新信息。
- `Continuity`：与已选片段是否连贯。
- `EventGain`：是否覆盖了新事件。
- `ViewStability`：推荐视角是否稳定，适合普通屏幕观看。
- `Redundancy`：是否和已选内容重复。
- `ViewJump`：推荐视角跳变是否过大。

### 2.4 浏览器观看轨迹与舒适度评价

在没有 VR 头显的条件下，系统先把普通浏览器作为 360°观看实验环境。播放器在用户自由拖拽、自动导览和摘要巡航三种模式下记录：

- 当前视频时间、观看 yaw/pitch/FOV。
- 当前模式：free / guided / summary。
- 推荐 yaw/pitch。
- 用户视角与推荐视角的角度误差。

页面进一步计算推荐导览路径本身的舒适度指标：

- 平均转向角。
- 最大转向角。
- 平均转向速度。
- 舒适度分数。

这样项目不仅回答“选哪些片段”，还回答“用户是否被平滑地引导到关键区域”。

### 2.5 S3-360-TourGuide 场景层

为了面向校园、景区和展厅导览场景，系统进一步加入 `S3-360-TourGuide` 场景层。它不是新的底层摘要算法，而是在 `S3-360-Guide` 的结果之上做应用化组织：

- 将摘要片段命名为“导览点”。
- 为每个导览点保存时间段、推荐 yaw/pitch、转向角、转向速度和舒适状态。
- 在 ERP 全景图上绘制导览路线地图，用编号点和箭头展示导览顺序。
- 计算综合导览分，融合事件覆盖、视角平滑度、内容多样性和摘要效率。
- 导出 `tourguide_report.md` 和 `tourguide_data.json`，便于答辩展示和后续实验记录。

这一步使系统从“算法复现 Demo”进一步变成面向真实沉浸式导览任务的完整原型。

## 3. 相比原方法的可讲点

原 S3-360 更关注摘要片段的选择，`S3-360-Guide` 更强调观看体验：

```text
原方法：选择关键片段
改进方法：选择关键片段 + 覆盖更多事件 + 控制视角跳变 + 输出更适合导览的视频
```

展示时可以这样说明：

> 本项目在原有显著性、重要性和多样性约束的基础上，进一步加入事件覆盖增益和视角稳定性约束，使生成的摘要不仅内容更丰富，而且观看过程更平滑，更适合普通屏幕导览。

## 4. 项目落地方式

代码中新增 `S3-360-Guide` 方法，并加入网页 Demo 的方法选择列表。用户可以直接在页面中切换 `S3-360` 和 `S3-360-Guide`，观察摘要片段、推荐视角框、时间轴和方法对比的变化。

网页播放器扩展了观看轨迹记录和 CSV 导出；页面新增“VR 导览轨迹与舒适度评估”面板，用推荐视角路径图和舒适度指标解释导览质量。该版本暂不依赖 VR 头显，便于课堂现场稳定演示。

在最新版本中，网页还新增了 `Scenario. S3-360-TourGuide` 主展示区。用户可以选择导览场景、填写外部地图参考链接，页面会直接展示导览点地图、综合导览分、导览点表和可下载报告。这部分用于突出课程项目的应用场景包装和工程完整度。

## 5. 适合答辩展示的总结

本改进不是简单增加界面功能，而是围绕 360°视频的观看特点加入导览约束。它解决了两个问题：

1. 摘要内容是否覆盖足够多的事件。
2. 摘要视频是否适合普通用户连续观看。

因此，`S3-360-Guide` 可以作为在原论文方法基础上的应用型创新点。
